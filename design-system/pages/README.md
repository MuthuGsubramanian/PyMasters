# Page-specific overrides

One file per page (`classroom.md`, `dashboard.md`, `login.md`, …) documenting only the
**deviations** from `../MASTER.md` — e.g. a page-specific layout pattern, an extra token, or a
relaxed density rule. Anything not stated here inherits from Master.

Retrieval rule for agents: when building page X, read `MASTER.md`, then `pages/x.md` if it
exists; the page file wins on conflicts.
